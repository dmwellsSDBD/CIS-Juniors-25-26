What is Mobile-First Design?

Start Small, Build Up

Mobile-First Approach:

  1. Start with mobile layout (smallest screen)
  2. Add tablet styles with media queries
  3. Enhance desktop styles with more media queries

Why Mobile-First?

  • ‚úÖ Forces you to focus on essential content
  • ‚úÖ Better performance on mobile devices
  • ‚úÖ Easier to add features than remove them
  • ‚úÖ Most users are on mobile anyway!

  • Emphasize: "We write CSS for mobile FIRST, no media query needed"
  • Explain: "Then we use media queries to add complexity for larger screens"
  • Time: 4 minutes

Media Queries: The Key to Responsive Design

What is a Media Query?

A media query applies CSS styles ONLY when certain conditions are met (like screen width).

/* Mobile styles - NO media query needed */ .card-container { grid-template-columns: 1fr; } /* Tablet styles - 768px and wider */ @media (min-width: 768px) { .card-container { grid-template-columns: 1fr 1fr; } } /* Desktop styles - 1024px and wider */ @media (min-width: 1024px) { .card-container { grid-template-columns: 1fr 1fr 1fr; } }
Presenter Notes:
  • Walk through syntax: @media (condition) { styles }
  • Emphasize: min-width means "at least this wide"
  • Point out: mobile needs NO media query - it's the default!
  • Time: 4 minutes

CSS Grid for Card Layouts

Why CSS Grid?

  • Perfect for 2D layouts (rows AND columns)
  • Simple, powerful syntax
  • Automatically handles spacing with gap property
  • Items flow naturally into grid cells

Essential Grid Properties:

.card-container { display: grid; /* Turn on grid */ grid-template-columns: 1fr 1fr 1fr; /* 3 equal columns */ gap: 20px; /* Space between items */ }
Understanding "fr" unit: 1fr = 1 fraction of available space
1fr 1fr 1fr = three equal columns
Presenter Notes:
  • Review: "Who remembers what display: grid does?"
  • Explain fr unit: "Imagine cutting a pizza into equal slices"
  • Show that gap replaces manual margin calculations
  • Time: 4 minutes

Responsive Images

The Problem

Images with fixed sizes break responsive layouts!

The Solution

.card img { width: 100%; /* Fill card width */ height: 250px; /* Consistent height */ object-fit: cover; /* Crop to fit, no distortion */ border-radius: 8px; margin-bottom: 15px; }
object-fit options:
  • cover - Fills space, crops excess ‚úì (best for cards)
  • contain - Fits entirely, may leave empty space
  • fill - Stretches to fit (can distort)
Presenter Notes:
  • Demonstrate what happens WITHOUT object-fit (distortion)
  • Show the difference between cover, contain, and fill
  • Explain why we need both width: 100% AND fixed height
  • Time: 4 minutes

Interactive Cards with Hover Effects

Make cards respond to user interaction

.card { background: white; border-radius: 12px; padding: 20px; box-shadow: 0 4px 6px rgba(0,0,0,0.1); transition: all 0.3s ease; /* Smooth animation */ } .card:hover { transform: translateY(-8px); /* Lift up */ box-shadow: 0 12px 24px rgba(0,0,0,0.2); /* Bigger shadow */ }
The transition property goes on the BASE element (.card), NOT on the hover state!
Presenter Notes:
  • Demonstrate the hover effect live in browser
  • Explain transform: translateY(-8px) means move UP 8 pixels
  • Show what happens without transition (instant, jarring)
  • Common mistake: putting transition on :hover instead of base element
  • Time: 4 minutes

Creating the Card Grid

Mobile Layout (Default)

.card-container { display: grid; grid-template-columns: 1fr; /* 1 column on mobile */ gap: 20px; max-width: 1200px; margin: 0 auto; }
Key Properties Explained:
  • max-width: 1200px - Prevents cards from getting too wide on huge screens
  • margin: 0 auto - Centers the container horizontally
  • gap: 20px - Creates consistent space between all cards
Presenter Notes:
  • Code this along with students
  • Have them wrap all 6 cards in a div with class="card-container"
  • Test in browser - should see 1 column of cards
  • Time: 5 minutes

Tablet Layout: 2 Columns

Using Media Query for 768px+

/* Tablet: 2 columns at 768px and wider */ @media (min-width: 768px) { .card-container { grid-template-columns: 1fr 1fr; gap: 25px; } .card button { width: auto; /* Button no longer full width */ } }
Remember: Tablet inherits ALL mobile styles, then adds or overrides!
Presenter Notes:
  • Add this code together
  • Test immediately - use DevTools to resize browser
  • Show that below 768px = 1 column, above = 2 columns
  • Point out we're ONLY changing what needs to change
  • Time: 5 minutes

Desktop Layout: 3 Columns

Using Media Query for 1024px+

/* Desktop: 3 columns at 1024px and wider */ @media (min-width: 1024px) { .card-container { grid-template-columns: 1fr 1fr 1fr; gap: 30px; } .card img { height: 200px; /* Shorter images for better proportions */ } }
Progressive Enhancement:
Mobile ‚Üí Tablet ‚Üí Desktop
Each level builds on the previous one!
Presenter Notes:
  • Add this final media query together
  • Test at all three breakpoints
  • Slowly resize browser to show smooth transitions
  • Explain: each breakpoint only changes what needs to change
  • Time: 5 minutes

‚ùå Mistake 2: Forgetting box-sizing

‚ùå Mistake 3: Fixed image widths
