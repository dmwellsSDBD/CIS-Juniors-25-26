JavaScript History & Background

1995

Created by Brendan Eich

Developed for Netscape Navigator to enable dynamic web pages that could respond to user events like clicks and hovers

1997

Standardized as ECMAScript

Ecma International created the ECMAScript specification. JavaScript is an implementation of ECMAScript, which continues to evolve with yearly releases

Interesting Fact: Brendan Eich called "ECMAScript" an "unwanted trade name that sounds like a skin disease" - it was a compromise between competing organizations!

JavaScript Today

Why JavaScript Matters

  • One of the most popular programming languages in the world
  • Supported by every major web browser
  • Powers modern web applications like Gmail and Google Maps
  • Used beyond browsers with technologies like Node.js for servers

How JavaScript Works

Interpreted Language

JavaScript code is executed by an interpreter without being compiled to machine language first

JavaScript Engines

Modern engines use Just-In-Time (JIT) compilation to convert JavaScript into faster executable code at runtime

Variables: Declaration & Assignment

Variable: A named container that stores a value in memory

Declaration

Use the let keyword to declare a new variable

let score;

Assignment

Assign a value to the variable using =

score = 2;

Initialization

Declare and assign a value on the same line

let maxValue = 5;

Identifiers & Naming Conventions

An identifier is a name created for items like variables

Rules for Identifiers
  • Letters, digits, underscores, or $ symbols
  • Cannot start with a digit
  • Cannot use reserved words (let, function, while, etc.)
Camel Casing Convention

Start with lowercase, capitalize subsequent words

✓ lastPrice
✗ LastPrice
✗ last_price

Constants

Constant: An initialized variable whose value cannot change
Declaration Syntax

Use the const keyword

const slicesPerPizza = 8;
Key Characteristics
  • Must be initialized when declared
  • Value cannot be reassigned
  • Use for values that won't change

Data Types: Dynamic Typing

JavaScript uses dynamic typing - variable types are determined at run-time

What This Means

  • Variables are not assigned a specific type when declared
  • Type is determined by the value assigned
  • Variables can be reassigned to different types

Example

let x = 5; // number
x = "test"; // string

The variable x started as a number, then was reassigned to a string - this is valid in JavaScript!

Common JavaScript Data Types

string

Group of characters in quotes

let name = "Naya";
number

Numbers with or without decimals

let score = 950;
boolean

True or false values

let hungry = true;
array

List of items in brackets

let teams = ["Broncos", "Cowboys"];
object

Collection of property-value pairs

let movie = {title:"Sing", rating:"PG"};
undefined

Variable not assigned a value

let message;
null

Intentionally absent value

let book = null;

Comments & Semicolons

Comments

Text for humans that JavaScript ignores

Single-line comment

// This is a comment

Multi-line comment

/* This is a
multi-line
comment */
Semicolons

JavaScript does not require semicolons at the end of statements

  • Only required when two statements are on the same line
  • Good practice: avoid multiple statements per line
  • Be consistent: use them or don't throughout your code

Input & Output in JavaScript

Getting Input

prompt() function

Displays a dialog box for user to enter text

let userName = prompt("Enter your name:");
  • Returns the text the user typed
  • Returns null if user presses Cancel
Producing Output

console.log() function

Displays text or numbers in the console

console.log("Hello, World!");
The Console: A location where text output is displayed. Accessible through browser development tools (F12)
